1. 깃 개념
원격 저장소(github)를 이용해서 협업에 있어 서로 간의 코드를 합치는(merge) 서비스가 가능함!
하지만 서로서로 작업한 내용을 원격저장소에 올리면 코드간에 충돌이 생길 수 있다!
그렇기 때문에 branch를 사용해서 원격저장소에 저장하기 전에 1차적으로 코드의 충돌유무와 에러를 확인할 수 있다

2. github 저장 순서
지역저장소(개인 컴퓨터)  ---(add + commit + push)-->  branch  ---(pull request)--->  원격저장소(github(main))

3. .git파일?
지역저장소에서 발생한 대부분의 내용들을 관리하는 폴더!
add를 하면 해당 폴더의 내용들이 .git으로 저장된다 이후 commit을 하면 해당 add에 대한 commit을 저장한다
마지막으로 push를 하면 git폴더에 담긴 내용들이 지정한 branch로 저장된다! (원격저장소는 보통 main 이라는 이름의 branch로 관리한다.)

4. branch?
branch는 원격저장소에 저장하기 전에 1차적으로 저장할 수 있는 임시 저장소 및 기록 저장소 라고 할 수 있다!
앞서 말했듯, 충돌을 피하기 위해 원격저장소에 branch를 생성하여 파일 및 코드를 관리한다.
하나의 원격저장소에는 다수의 branch를 생성할 수 있으며, 프로젝트마다 개개인의 branch를 생성하고 관리(우리 조 처럼!) 하거나
기능마다 branch를 생성하는 방법도 있다. push를 하면 branch에 .git에 담긴 내용들(수정사항, commit 등)이 저장된다.
일회성으로 사용하고 branch를 버리는(삭제) 경우가 많다!
